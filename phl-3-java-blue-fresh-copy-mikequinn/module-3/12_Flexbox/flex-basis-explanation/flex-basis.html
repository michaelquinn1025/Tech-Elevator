Flex-Basis Explained

To understand the flex-basis property, we need to understand how flex-grow and flex-shrink work.

For these examples we will be working with these classes:

            
    .flex-example-1 {
        border: 3px black solid;
        display: flex;
    }

    .box-flex {
        background-color: blueviolet;
        color: white;
        text-align: center;
        width: 100px;
        padding: 15px 5px;
        margin: 1px;
    }

The examples we are going to look at, we will be working with a div similar to this:

            
    <div class="flex-example-1">
        <span class="box-flex">A</span>
        <span class="box-flex">B</span>
        <span class="box-flex">C</span>
    </div>

This HTML gives us the results below - it renders each box at it's true size (100px).

A B C

If we add flex-grow: 1 to the span containing B, it will stretch to fill all the extra space.

A B C

If we also add flex-grow: 2 to the span containing C, it will stretch to twice the amount B stretches:

A B C

If we remove the flex-grow: 1 from the span containing B and leave the flex-grow: 2 in the span containing span C, span B will no longer be allowed to grow and will shrink down to it's original size:

A B C

We can use the flex property as shorthand for flex-grow, flex-shrink, and flex-basis properties. This will help us understand flex-basis.

In the property flex: 1 0 200px, 1 indicates the flex-grow value, 0 indicates the flex-shrink value, and 200px indicates the flex-basis value.

If we leave the span containing C as above (flex-grow: 2) but add flex: 1 0 200px to the span containing B, we are saying that B is allowed to grow but the flex-basis specifies that B must take up AT LEAST 200px. As you can see, C is no longer 2 the times the amount B stretches because if B is at least 200px, there isn't room for C to grow more than it is. In the case that flex-basis is specified for an element with flex-grow, the flex-basis value indicates that the element must be at least the size specified by flex-basis but can grow larger if need be. The math of how the proportions are calculated under this circumstance are complex but the main point here is that B has a minimum size.

A B C

If we leave B as above (flex: 1 0 200px) and modify C to have no longer have a flex-grow value, but rather to have flex-shrink: 1 (flex: 0 1) we will see that it no longer grows and, thus, shrinks back down to it's original size while B stretches to fill the rest of the space.

A B C

However, if we do the same and specify a flex-basis of 50px for C (flex: 0 1 50px) we will see that C shrinks and will no longer be allowed to grow to be more than 50px. In the case that flex-basis is specified for an element with flex-shrink, the flex-basis value indicates that the element will shrink and cannot be larger than the flex-basis value.
